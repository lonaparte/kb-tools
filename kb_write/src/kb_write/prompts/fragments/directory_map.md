| Path                            | Contents                                              | Agent may write?    |
|---------------------------------|-------------------------------------------------------|---------------------|
| `papers/`                       | Per-paper md from Zotero (incl. book chapters at `<KEY>-chNN.md`) | AI zone only        |
| `topics/standalone-note/`       | Zotero standalone notes (v26; was `zotero-notes/`)    | AI zone only        |
| `topics/agent-created/`         | AI-generated topic syntheses (v26; was top-level `topics/`) | Full file           |
| `thoughts/`                     | Dated thought notes (`YYYY-MM-DD-*.md`)               | Full file           |

### Multi-md papers (v26)

Books and long theses processed by `kb-importer import papers --fulltext`
produce several sibling paper mds that share one Zotero key:

```
papers/BOOKKEY.md         # whole work (parent)
papers/BOOKKEY-ch01.md    # chapter 1  ← shares zotero_key = BOOKKEY
papers/BOOKKEY-ch02.md    # chapter 2
...
```

All chapters are `kind: paper` with `item_type: book_chapter`. Use
the MCP tool `list_paper_parts(zotero_key)` to enumerate every part
of a split work. `find_paper_by_key` still returns only the
whole-work md.

### Deprecated v25 paths (NOT indexed)

| v25 (not indexed in v26)                    | v26 replacement                   |
|---------------------------------------------|-----------------------------------|
| `zotero-notes/<KEY>.md`                     | `topics/standalone-note/<KEY>.md` |
| `topics/<SLUG>.md` (top-level)              | `topics/agent-created/<SLUG>.md`  |
| `thoughts/<date>-<KEY>-chNN-*.md`           | `papers/<KEY>-chNN.md`            |

Move, rewrite, or delete files at legacy paths — v26 does not
auto-migrate. `kb-mcp index-status` flags them.
